Getting Started

LabWired transforms physical hardware constraints into programmable AI-ready assets. Follow these three steps to build your first autonomous hardware agent.

1. Ingest Datasheet

Use the LabWired AI tool to extract register maps and behavioral logic directly from PDF datasheets.

labwired_ai.main ingest-datasheet firmware/manual.pdf

2. Verify Model

Run the automated truth-checker to ensure the generated model matches physical constraints.

python ai/tests/e2e_test.py --peripheral peripheral.yaml

3. Run Emulation

Execute your firmware against the bit-accurate digital twin in a fully deterministic environment.

labwired sim run --config system.yaml

Technical Architecture: AIPi Protocol

The Agentic Interface for Peripheral Ingestion (AIPi) is our core protocol. It allows LLMs and automated verifiers to operate hardware as a programmable service.

AI Generation & Grounding

Unlike standard SVD files, LabWired AI models include grounding metadata. This provides machine-readable evidence for every register and transition, eliminating hallucinations.

"LabWired is a deterministic back-end. If an agent provides an incorrect system definition, the simulation fails with a BusPanic. This enforces total reliability for autonomous agents."
